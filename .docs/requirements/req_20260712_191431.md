# Pokédex — Documento de Requisitos

**Data:** 2026-07-12 19:14  
**Versão:** 1.0  
**Fonte:** PokeAPI v2 (https://pokeapi.co/api/v2)

---

## 1. Visão Geral

Aplicação front-end *single-page* em Vanilla JavaScript que consome a PokeAPI v2 para exibir um catálogo navegável de Pokémon. O usuário pode listar, buscar, filtrar e visualizar detalhes de cada Pokémon sem recarregar a página.

## 5. Critérios de Aceite

### CA-01: Listagem Inicial
- Dado que o usuário acessa a página, então 20 Pokémon são carregados automaticamente.

### CA-02: Load More
- Dado que o usuário clica "Carregar mais", então os próximos 20 Pokémon são adicionados ao final da lista.

### CA-03: Busca
- Dado que o usuário digita "char" e submete, então todos os Pokémon com "char" no nome são exibidos.

### CA-04: Filtro por Tipo
- Dado que o usuário seleciona "Fire", então apenas Pokémon do tipo Fire são exibidos.

### CA-05: Detalhes
- Dado que o usuário clica em um card, então a view de detalhes do Pokémon é exibida com sprite, stats, tipos, habilidades, peso e altura.

### CA-06: Cache
- Dado que a aplicação já buscou uma página, então uma segunda requisição idêntica não atinge a rede (exceto se o cache expirou).

### CA-07: Tratamento de Erro
- Dado que a API está indisponível, então uma mensagem legível é exibida ao usuário.

### CA-08: Responsividade
- Dado que a viewport tem 375 px de largura, então o layout se adapta sem quebra ou overflow horizontal.
