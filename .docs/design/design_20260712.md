# Design e Topologia: Atualização do README.md

## 1. Topologia do Sistema Alvo
- **Entidade:** `README.md` (Raiz do repositório)
- **Formato Arquitetural:** Documento estruturado em Markdown (`.md`)

## 2. Contratos de Estrutura (Interfaces Visuais)
O arquivo `README.md` deve implementar rigorosamente a seguinte estrutura de seções (H2):

```markdown
# Gerenciador de Workflows de Agentes

(Texto introdutório e Descrição do Projeto - com revisão para alinhar ao unificado /requirements)

## Requisitos Pendentes
- [Texto descrevendo pendências e solicitações abertas identificadas]

## Workflows Disponíveis
- Requirements ([requirements.md](file:///d:/gen/.agents/workflows/requirements.md))
- Code ([code.md](file:///d:/gen/.agents/workflows/code.md))
- Test ([test.md](file:///d:/gen/.agents/workflows/test.md))
- Explore ([explore.md](file:///d:/gen/.agents/workflows/explore.md))
- Commit ([commit.md](file:///d:/gen/.agents/workflows/commit.md))

## Comandos e Uso
- `/requirements`
- `/code`
- `/test`
- `/explore`
- `/commit`
```

## 3. Schemas de Substituição (Diff Abstrato)
* **Remoção de Nós Antigos:**
  - Node: `req.md`
  - Node: `design.md`
  - Node: `/req`
  - Node: `/design`
* **Inserção de Nós Novos:**
  - Node: `requirements.md`
  - Node: `/requirements`
  - Node: `## Requisitos Pendentes`

## 4. Assinaturas de Validação
```go
// Representação abstrata do contrato de atualização
type DocumentoReadme interface {
    PossuiSecao(nomeSecao string) bool // Deve retornar TRUE para "Requisitos Pendentes", "Workflows Disponíveis", "Comandos e Uso"
    ListaComandos() []string // Deve conter apenas: /requirements, /code, /test, /explore, /commit
    ListaArquivos() []string // Deve conter apenas: requirements.md, code.md, test.md, explore.md, commit.md
}
```
