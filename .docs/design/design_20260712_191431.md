## 1. Arquitetura Geral

Padrão **Clean Architecture** adaptado para front-end com camadas concêntricas:

```
┌─────────────────────────────────────┐
│           VIEW (DOM)                │  Renderização e eventos do usuário
├─────────────────────────────────────┤
│        CONTROLLER (Orquestrador)    │  Casos de uso / lógica de navegação
├─────────────────────────────────────┤
│         SERVICE (Aplicação)         │  Orquestração de dados, cache
├─────────────────────────────────────┤
│      REPOSITORY (Infraestrutura)    │  Acesso à PokeAPI v2
├─────────────────────────────────────┤
│          DOMAIN (Entidades)         │  Modelos de negócio puros
└─────────────────────────────────────┘
```

**Direção das dependências:** Sempre para dentro. Camadas internas não conhecem camadas externas.

## 6. Fluxos de Dados

### 6.1 Listagem Inicial

```
User → View.renderList()
     → Controller.initialize()
     → Service.getPage(0, 20)
     → Repository.fetchPage(0, 20)
     → Cache.has('page_0_20')? → Cache.get : fetch(PokeAPI)
     → Service → Controller → View.renderList(pokemonList, false)
```

### 6.2 Busca por Nome

```
User input → View.bindSearch(query)
           → Controller.onSearch(query)
           → Service.searchByName(query)
           → Repository.fetchByName(query)
           → Service → Controller → View.renderList(results, false)
```

### 6.3 Filtro por Tipo

```
User select → View.bindFilter(typeName)
            → Controller.onFilter(typeName)
            → Service.getByType(typeName)
            → Repository.fetchByType(typeName)
            → Service → Controller → View.renderList(results, false)
```

### 6.4 Detalhes do Pokémon

```
User click card → View.bindPokemonSelect(id)
                → Controller.onSelectPokemon(id)
                → Service.getDetail(id)
                → Repository.fetchById(id)
                → Service → Controller → View.renderDetail(pokemon)
```

### 6.5 Load More

```
User click → View.bindLoadMore()
           → Controller.onLoadMore()
           → Service.getPage(currentOffset + 20, 20)
           → Repository.fetchPage(...)
           → Service → Controller → View.renderList(newPokemon, true)
```

## 7. Módulo de Cache

```
CacheStrategy: write-through
TTL padrão: 1.800.000 ms (30 min)
Chave: `${endpoint}_${params}` → ex: "pokemon_0_20"
Armazenamento: window.localStorage
```

## 8. Tratamento de Erros

```
Repository
  └─ fetch falhou → lança PokemonError(code, message)

Service
  └─ captura PokemonError → retorna Result.Err(error)

Controller
  └─ Result.Err → View.renderError(mensagem amigável)

View
  └─ renderError → exibe <div class="error"> com texto e botão "Tentar novamente"
```
